# ¿Porque decir distribuciones `GNU/Linux`?

![GNU/Linux](https://external-content.duckduckgo.com/iu/?u=https%3A%2F%2Fcodelearn.com%2Fwp-content%2Fuploads%2Fsites%2F5%2F2022%2F10%2Fgnu-linux.jpg&f=1&nofb=1&ipt=4cc8f0457e2c85c70a375bfac31747f444fa80fe7637391680292ecaa02da123)

Vi que algunas usuarios no les gusta del todo que se diga *distribuciones `GNU/Linux`*, en vez de distribuciones Linux. Bueno no me extenderé mucho y tratare de argumentar porque esta bien decir `GNU/Linux`. 

Que un sistema operativo utilice `GNU` + `Linux`, ya nos asegura que dicho sistema operativo es transparente en sus principales funciones, y en su source code.

El `GNU` indica **software libre**. Usar `GNU` incita a respetar las reglas del software libre. Cualquier OS que use GNU modificado, tiene que estar ligado a las premisas mencionadas en `GPL License`.

Un sistema operativo puede tener kernel Linux y a su vez no usar GNU. **Android** es el ejemplo perfecto, no usa GNU, solo Linux. Que use GNU implicaría ceder a la licencia GPL, y eso no lo quiere google.

Linux se contemplo para GNU. GNU usa Linux como su kernel mas común. Linux es mas independiente, este se encarga de comunicarse con el hardware, la BIOS, y etc.

Ahora bien, uno cuando dice distribuciones linux, no piensa en android, piensa en debian, ubuntu, arch, y etc. Distribuciones que usan GNU y Linux como base.

En la actualidad **Linux esta en todos lados**, **GNU** también, pero no tanto (por su licencia de uso). Yo por eso considero que es bueno decir GNU/Linux, separa el OS mas pro comunidad, sobre OS mas corporativos.

- [Filosofía del software libre](https://www.gnu.org/philosophy/free-sw.es.html)
- [GNU Generic Public License (GPL)](https://es.wikipedia.org/wiki/GNU_General_Public_License)

## Conclusión
Sistema operativo `GNU/Linux` significa software libre como base. Y el software libre también es una filosofía de desarrollo.

No veo mal que digan distribuciones Linux, pero tampoco veo mal el mencionar a GNU. GNU es software, y filosofía de desarrollo.

> En general los argumentos que daban contra de decir `GNU/Linux` no me parecían buenos, eran tipo: *"`GNU/Linux` es mas largo y feo"*, *"solo lo dicen los creídos"*, o *"nadie le dice así"*.
